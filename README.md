## ğŸ“– About The Project

This repository contains the complete data analysis pipeline used to demonstrate systematic noise mitigation during spin shuttling in a linear $^{28}$Si/SiGe quantum dot device.

Mobile spin qubit architectures promise flexible connectivity for quantum error correction, but preserving coherence during transport is a major challenge. This codebase supports the findings that **noise mitigation strategies**â€”specifically micromagnet design optimization, motional narrowing, and dressed-state drivingâ€”can extend the coherence time of mobile qubits by nearly an order of magnitude, reaching over $30~\mu s$.

The pipeline handles raw HDF5 experimental data, performs curve fitting (Ramsey, Echo, Rabi), extracts coherence parameters ($T_2^*$, $\alpha$), and generates the publication figures.

### Stage 1: Data Processing

Located in `code/get_processed_data/`. These notebooks:
- **Load raw experimental data** from HDF5 files (via `common_scripts.raw_data_loader`)
- **Process and analyze** the data (fitting, parameter extraction, simulations)
- **Save processed data** to `code/processed_data/` folder (as `.pkl` and `.json` files)

**Key notebooks:**
* **`fig23_process_stationary.ipynb`**: Processes stationary qubit measurements, extracts T2 coherence times
* **`fig4_process_shuttling.ipynb`**: Analyzes coherence during periodic shuttling to demonstrate motional narrowing
* **`fig5b_process_LZSM.ipynb`**: Processes Landau-Zener-StÃ¼ckelberg-Majorana interference data
* **`fig5c_process_Rabi.ipynb`**: Extracts Rabi decay times for the dressed-state shuttling protocol
* **`appendix_*.ipynb`**: Contains numerical simulations for theoretical models (Two-point DC model, Coarse-grained model, LZSM)

**Note:** This directory ONLY processes data - it does NOT generate plots.

### Stage 2: Figure Generation

Located in `code/get_plots/`. These notebooks:
- **Load processed data** from `code/processed_data/` folder (generated in Stage 1)
- **Generate publication-quality figures** using standardized plotting utilities
- **Save PDF figures** to `code/figures/` folder

**âš ï¸ Important:** This is the **ONLY** directory where plots for the paper are generated.

**Key notebooks:**
* **`main_text_figures.ipynb`**: Generates all main text figures (Figures 2, 3, 4, 5b, 5c, and related panels)
* **`appendices_figures.ipynb`**: Generates supplementary and appendix figures

All figures use consistent styling and color schemes defined in `common_scripts.plot_utils`.
